## 当前优先架构假设（已定稿范围，变更时同步续篇 5 与决策记录表）

- **部署**：**境外轻量云主机**对外提供 **HTTPS**；**域名可在国内注册商购买**，DNS **A/AAAA 解析到境外 IP**（与「网站托管在大陆机房需 ICP 备案」不是同一套流程，具体以注册商与监管要求为准）。
- **客户端形态**：**学生必须安装** **Win / Mac 桌面端**；**教师使用纯 Web 管理台**（浏览器访问，**不强制**装学生端同名的桌面版）。后端对外 **HTTPS API** 同时服务**学生端与教师 Web**。
- **教师身份（无注册流）**：**教师不进行账号注册**；**首次部署/初始化** 在 **Web 管理台** 或 **安装向导** 中**设置一次「管理员/教师」登录密码**（可后续在已登录态下修改），凭此密码管理全班。**课堂场景、不涉及高价值隐私或资金资产** 的前提下，**不把「教师账号被盗」按高优先级威胁建模**（仍使用 **HTTPS**；若后续有更高要求可再加 2FA 等，非本期必项）。
- **角色**：**业务上** 仅 **学生** 与 **教师**；**教师** 即 **Web 管理端操作者**（无「教师用户表多行注册」的复杂流程，除非你在 `design` 里为多教师扩展预留）。
- **作业与课程**：**一章一份作业**；**全班同一题目**；**无限次重做**；**每次新提交覆盖存储，只保留该生该章的「当前最新」一份**（无历史版本时间轴；若需要「最近一次提交时间」可仅存元数据，见 `design`）；**不做防抄袭**；**无转班**业务。
- **身份与学号防错号**：**学号+姓名 由教师提前在 Web 管理台导入**（或批量上传）；**学生首次注册/绑定** 时**必须**输入**与名单完全一致的 学号 与 姓名**（**任意一项不符则注册失败**），以此防止输错、冒用；**找回密码不做邮箱/手机验证**；**学生密码采用可逆存储**（经服务端加密保存），**教师界面可直读/复制明文** 以便课堂处理；**教师可重置任意学生密码**。
- **LLM Chatbot**：采用**尽可能低成本的接入**——**国内** **OpenAI-兼容/各家 HTTP API 三选一 首发 暂缓决策**，**做到集成该能力时再选型**（候选：阿里云通义、智谱 BigModel、百度文心等，**以当时官网与合规为准**）；**不做 RAG**，对话以**当前练习上下文 + 可选系统提示**为主。密钥以**环境变量或部署配置** 注入。
- **Notebook / 代码执行**：**默认**走 **JupyterLite / Pyodide**（见 §5）；**当课程需要**「**联网自行 `pip install` 装 PyPI 上任意包**」**或** 依赖 **C 扩展/非纯 Python 包** 等 Pyodide 难覆盖能力时，**切到 本机 CPython 子进程** + **可写 venv/用户目录** + **允许出站 `pip`（在客户端侧执行，pip 全开放，见 §5）**；**超时、资源上限、工作目录、supply-chain 风险** 在 `design` 中写清。
- **`pip` 策略**：**全开放** —— 学生可对 **PyPI（或所配 index）** **自行 `install`**，**不做** 教师**白名单** 为默认策略（`design` 中写清单与建议「课堂用」；若日后要加黑名单再迭代）。

# 前篇：整体准备清单（十段）

## 1. 产品与教学设计（非代码）

- **用户与场景**：**学生、教师**两角色；**教师** 用 **Web** 建课、维护名单、发作业、批改、密码与内容管理；**学生** 用**桌面端** 学习与提交。
- **课程结构模型**：**章**；每章有 **教学内容** + **该章唯一一份作业**；全班题目相同；学生可**无限次重交**；**同一学生、同一章：只保留最新一次提交物**（覆盖写、不保留历史版本列表；若需「最后提交时间」仅存元数据，见 `design`）；**不实现转班、不实现防抄袭**。
- **名单与注册**：**教师** 在 Web 端 **预导入 学号+姓名**；**学生** 在桌面端 **自注册/首次绑定** 时 **严格匹配** 名单，**不匹配则无法通过**；**无**「学生随便填、教师后审」的宽松模式（除非你后续改产品）。
- **账号标识（桌面端）**：**学号** 与名单键一致；**桌面端** 本地草稿与**联网提交**、失败重试策略。
- **练习与执行**：见 §5；由平台**设定**单会话/CPU/内存与 **pip/网络** 策略。
- **评价与成绩**：以**教师给分+评语**为主；成绩单导出、迟交等规则在 `design` 中**按需**收敛。

## 2. 唯一产品形态：学生桌面 + 教师 Web + 境外 API

- **学生端**：本机 **Win/Mac 应用**（**必装**）；线下可编辑，联网 **HTTPS 提交**作业，按 **学号** 与 **章** 关联。
- **教师端**：**纯 Web 管理台**（**浏览器**）；**不强制** 安装与学生相同的桌面版；**无「教师注册」** —— 仅**管理员密码** 登录后操作；需支持：**名单导入、章节内容、作业题面、按学号查看** **当前** **提交、打分、查看/重置学生密码**（**无「历史每次提交」列表**，与「只保留最新」一致）。
- **大模型数据流**：题面与学生提问是否发往模型、对话是否**落库**、教师是否**审计对话**，在**隐私与校务**条款中写清；数据存**境外**时与**教育数据、出境**要求一并评估。

**典型逻辑分层**：学生 **桌面** → **TLS** → **API**；教师 **浏览器** → **同一 API / 同域 Web**；**关系型/对象存储**；可选 **JupyterLite 内嵌** 或 **本机 CPython 子进程**；**Chatbot** 经**国内 API 网关**（见 §6）；作业与元数据落库。

## 3. 账号与权限（无微信/QQ、无独立管理员）

- **身份模型**：**学生** 为**注册+登录** 主体；**教师** 为 **Web 管理端操作者** —— **不设「教师用户注册」**，**单（或多）个管理员密码** 进入管理台（`design` 中定 **密码存储**：哈希即可，**不** 要求与「学生可逆」同型）。`Role` 在**接口层** 仍可区分 `student` / `teacher_session`。
- **凭据与注册**：**学号+姓名 预导入** + **首登严格匹配**（见上）；**学号+密码** 用于学生日常登录；**无** 邮箱/手机作为找回通道；**学生密码** **服务端可逆加密存储**，**教师端可查看（解密展示）/重置** 学生密码。课堂场景、**不** 将「教师单密码泄露」 按**资产级** 事件做一期专项（**HTTPS** 仍上）。
- **不使用** OAuth 与微信/QQ 登录。

## 4. 教学内容与作业子系统

- **内容管理**：章级 **Markdown/富文本**、代码块与附件；**仅教师** 可编辑发布（Web）。
- **作业系统**：**每章一题、全班同题**；**无限重做**；**只保留** 每生每章**最新** 提交；提交物规格在 `design` 中定；**教师 Web** 批改；学生桌面 **离线队列** 定义清楚（重交通道与「仅最新」 一致，**不** 积累本地历史链除非另有同步需求）。

## 5. Notebook / 代码执行：JupyterLite 与本机 CPython 的分界

- **目标**：在**小体量教学**下**最快落地**；**学生联网可自行 `pip install`（对 PyPI）** 时，以 **本机 CPython 子进程** 为主能力承载，**JupyterLite/Pyodide** 为**轻量默认/补充**（适合标准库+少数 micropip 场景）。

**何时用 JupyterLite / Pyodide（优先简单、无系统级 pip 依赖）**  
- 以 **标准库**、教学脚本为主，**不依赖** 需从 PyPI **任意**拉取的**重型/二进制** 包。  
- 或作为 **入门章节** 的快速演练层（**启动快、包体可预测**）。

**何时切到 本机 CPython 子进程（联网 pip）**  
- **需要** 学生执行 `pip install <包名>` 安装**任意**（或**课程指定列表外**）的 **PyPI 包**；或依赖 **C 扩展、非 wheels 在 Pyodide 不可用** 的库。  
- 实现上：**内嵌/随包携带** 某版本 **CPython**、**每生/每作业 venv 或 用户级 site-packages 目录**、**允许 HTTPS 出站到 PyPI 与** 选定的 **index URL**、**可配置镜像**（如清华/阿里源）、**记录安装日志** 便于排错。

**`pip` 与策略（已定）**：**全开放** —— **默认不** 做教师侧 **包/版本白名单**；学生可对 **公网 index** 执行 `install`；**恶意包/供应链** 通过**课堂说明 + 安装日志** 管控，**可选** 日后在**同清单** 另起迭代加「黑名单」。

**共同底线**（`design` 中量化）：**子进程/容器化隔离**、**工作目录**、**CPU/内存/单会话时长**、**子进程** **kill**、**供教师/助教可见的安装记录**（**便于排错**）；**JupyterHub 集群** 仍**不作为** 一期目标。

## 6. LLM Chatbot 子系统（免费优先、无 RAG）

- **产品范围**：**不建向量库、不做 RAG**；上下文以**当前题/当前章节 + 用户当轮输入** 为主；是否附带**代码片段** 在 UI 中可选。
- **「免费」与供应商**：**国内** **HTTP API 哪家首发 暂缓** —— **做到集成该能力** 时再从 **通义、智谱、文心** 等中选一家；届时再读**免费/试用** 档与**合规**；超预算时按量/限流/降级。清单**不** 锁死**首发厂商**。
- **技术**：**HTTPS 出站** 至所选 API 域名；**流式/非流**、**限流**、**关键审计**、**防注入**、费用监控；**密钥** 经**环境变量/部署配置** 注入，**不** 写入仓库。

## 7. 工程与基础设施

- **代码与协作**：单仓优先；**CI**；**学生** **Win/Mac 构建与代码签名**；**教师** **Web 前端 + 同仓后端**。
- **运行与部署**：**境外轻量云**、**Let’s Encrypt** 或云证书、**WAF/安全组** 仅开必要口；**关系型库** 存用户、**预导入名单**、章、作业、提交、**注册绑定状态**、**可选**对话摘要。
- **域名**：国内购域 + 解析境外；**备案** 与**大陆托管** 关系见假设节。
- **监控告警**：**API 错误、LLM 429/欠费、学生端崩溃率**（若可采集）。

## 8. 合规、安全与隐私

- **隐私与校务**；**未成年人** 如适用；**教育数据** 与**境外机** 存储的告知与同意；**学生自行 pip** 属 **供应链** 与 **本机** 边界，在**课堂说明** 中写出；**pip 全开放** 时尤需**可观测** 安装记录。
- **学生密码可逆 + 教师可读**：**课堂使用、不涉及高价值资金资产** 场景下，**不** 把**教师端单密码** 当**高等级** 威胁**单独** 立项；仍使用 **HTTPS**、**可逆** 与**管理密码** 在 `design` 中写清**加解密** 与**机架侧** 控制；**学生 pip** 以**教师可见** 安装日志 为主 排错手段。

## 9. 团队与项目治理

- **角色**：产品、教学、**学生客户端**、**教师 Web**、后端、**运维** 可兼岗。
- **预算**：**境外主机**、域名、**LLM 超额** 缓冲、**学生端代码签名**；**大集群 Notebook 云** 非一期必项。

## 10. 已定产品分叉（已写入上文；`design` 中落到接口与表结构）

- **教师** **仅 Web**、**不注册** —— **管理员密码** 进入；**学生** **必装桌面**；**教师** **用浏览器** 完成全部管理（**不装学生端**）。**不** 将**教师机密码泄露** 按**高优先级** 与**资金级** 事件平行建模（**课堂** 前提）。
- **JupyterLite** 与 **本机 CPython+联网 pip** 的**选型**：见 **§5**；`design` 中列出 **课纲依赖** 与**执行栈** 切换点；**pip 全开放**（§5）。
- **学号+姓名** **仅教师预导入**；**学生** **首登** **必须** 与名单**完全一致**。
- **学生密码**：**可逆**（**教师** **可见/复制** 明文 展示）；**管理员密码** 在 `design` 中 另述（通常 **bcrypt 等 不可逆** 即可）。
- **重做**：**只保留** 每生每章**最新** 一次提交；**不** 做**历史** 时间轴 列表（**可** 存 `submitted_at` 元数据）。
- **国内 LLM 首发**：**暂缓**，**接该能力** 时 再 选型。
- **`pip`**：**全开放**（§5）。

# 续篇：分阶段、MVP、风险、待澄清

## 续篇 1. 建议的交付阶段（从准备到可运营）

| 阶段 | 目标 | 依赖准备（摘要） |
|------|------|------------------|
| M0 概念验证 | **名单导入** + 学号**匹配**注册 + **HTTPS** + 一章一作业 + **一次** Chatbot（**API 厂商** 可 **mock** 或 **暂缓接国内**）+ **JupyterLite 或** **CPython+pip 最小闭环**；**教师** **仅** **管理员密码** **无注册** | 境外云、域名、TLS、学生端 Hello 包、**教师 Web** 初始化；**国内 LLM 密钥** **做到该步再配** |
| M1 教学端 MVP | **多章**、**无限重做**、**教师打分**、**看/设密码/重置**、**pip+CPython 路径** 按课纲能跑通 | 库表、教师权限、**审计**、**名单与绑定** 状态机 |
| M2 体验加固 | 学生端更新、**限额/超时/ pip 镜像** 打磨、**LLM 欠费** 策略、**日志与备份** | 运营手册 |
| M3 可选加强 | **更强沙箱**、**对 pip 的治理**（黑名单/审批）若教学需要 | 不扩 RAG、不上 JupyterHub 除非**明确**需要 |

**分期原则**：**别在一期** 堆 **JupyterHub 云**；**先** 跑通 **名单 + 学生端 + 教师 Web + 一条能 pip 的 CPython 链**（若课纲需要）。

## 续篇 2. MVP 与「全量」的边界

- **MVP 必含**：**学生** 注册/登录 + **教师** **无注册**、**管理员密码**、**预导入+匹配注册**、**一章一作业、同题、无限重做**、**只保留最新提交**、**学生桌面 + 教师 Web**、**学生密码可逆+教师可见/重置**、**HTTPS**、**pip 全开放** 下的 **CPython 路径**（课纲有要求时）+ **JupyterLite 可选**、**无 RAG** 的 **Chatbot**（**国内 API 首发** **可** 与 **M0/M1** **并行** 再定）。
- **刻意不做**：**微信/QQ**、**防抄袭**、**转班**、**多管理员**、**RAG**、**云端 JupyterHub 教学**（一期）。

---

## 续篇 3. 技术选型 spike（轻量）

- **学生** **Electron 或 Tauri** + 后端 + **教师 Web** 同域联调；**教师** **管理员密码** **初始化** 流程。
- **JupyterLite** 一屏 + **本机** **CPython** **子进程+venv+pip** 一屏，各跑通同一章节的**依赖差异** 样例（如「仅标库」vs「需 `pip install numpy`」）；**pip 全开放** 路径 跑通。
- **国内 LLM**：**实施到 Chatbot 集成** 时再 **选定** **1 家**、读**免费额度、QPS**；写 **fallback** 文案；前期 **可 mock**。

## 续篇 4. 风险登记

| 风险 | 说明 | 缓解 |
|------|------|------|
| 教师**单密码** **泄露** | **课堂** 场景以**教学** 为主，**不** 按**资金级** 事件建**完整** 威胁模型 | **HTTPS**、**机房/教室** 保管**登录页**、**可** 在**后续** 迭代加 **2FA** |
| 学生 **pip** | **恶意/投毒包**、**本机** 与**供应链** 风险 | **全开放** 下 **安装日志**、**课堂规范**、**可选** 日后 **黑名单** |
| 免费 API 不可持续 | 额度用尽、变价 | 用量与费用监控、**学生侧** 明确提示 |
| 数据在境外机 | 学号、作业、对话 | 告知、校务、**最小** 留存 |
| 名单**泄露** 或**导入错行** | **错绑** 或**冒名** 风险 | **导入校验**、**样例** 对账、**首登** **严格** 学号+姓名（已定） |

## 续篇 5. 已决项 与 仍留给 `design` 的细目

**本清单 已 拍板，不再 作为 开放问题 讨论：**

1. **教师**：**不注册**；**管理员密码** 初始化；**不** 将 **教师机密码 被盗** 按 **高优先级** 资金级 事件（**课堂** 前提，见 §8）。
2. **学生密码**：**可逆**（**服务端** 加密 **存储**，**教师** **界面** 可 **看** **明文/复制**）。**具体** 算法、**KMS/机架 密钥**、**轮换** 在 `design` 中写清。
3. **重做**：**只保留** 每生每章 **最新** 一次 提交 内容；**可** 存 `submitted_at` **等** 元数据。
4. **国内 LLM 首发 厂商**：**暂缓**，**做 Chatbot 集成** 时 再 从 **通义/智谱/文心** 等 中选 并 读 当时 定价与合规。
5. **`pip`**：**全开放**（§5），**不** 做 **教师 白名单** 为 一期 默认。

**仍 由 `design` 写细**：**管理员密码** 的 **bcrypt/argon2** 等参数；**学生密码** 的 **可逆 加密 方案** 与 **密钥 存放**；**提交物** 文件格式与 **仅保留最新** 的**数据库/存储**约束（含是否保留 `submitted_at`）。
