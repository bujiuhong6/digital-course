# AI + Python 教学系统 — 准备清单（完整：前篇 + 续篇）

**文档角色**：在 `vendor/superpowers/skills/brainstorming/SKILL.md` 工作流下，将大系统拆成可独立推进的子域。前篇列「要准备什么」；续篇列分期、MVP、风险与流程衔接。正式架构规格见后续 `*design.md`，实现任务见 `writing-plans`。

**产品定位（本轮已定）**：系统用于**学生 Python 练习与自助过关**；**不**建设独立「作业/打分/批改」子系统；**教师**不在系统内**打分**，不以「批作业」为必备工作流。教师通过 **Web** 编辑章节、维护名单、**可**查看学生各章是否完成/进度，**不**做成绩单业务。

**章节内容来源与生成方式（已定）**：**各章原始材料**（讲义、大纲、参考链接、视频要点等）**由本人（项目唯一维护者）后续提供**；进入建设阶段后，用 **AI**（与实现时选定的国内/境外大模型能力一致，**以 `design` 与工程约束为准**）**自动生成**每章内的**核心知识点**、**引导练习**（代码区可带可执行提示）与**扩展练习**（题面只给目标/期望现象、**不**在代码区生成完整可抄标答）。**教师**在 **Web** 端对 **AI 草稿** 做**审阅、删改、锁定**后再发布；**人审**为**必过**环节，**不**承诺完全无人参与上线。

## 当前优先架构假设（已定稿范围，变更时同步续篇 5 与决策记录表）

- **部署**：境外轻量云主机对外提供 **HTTPS**；域名可在国内注册商购买，DNS **A/AAAA** 解析到境外 IP（与「网站托管在大陆机房需 ICP 备案」不是同一套流程，以注册商与监管要求为准）。
- **客户端形态**：学生**必须**安装 **Win / Mac 桌面端**；教师使用**纯 Web 管理台**（浏览器）。后端 **HTTPS API** 同时服务学生端与教师 Web。
- **教师身份（无注册流）**：教师**不进行**账号注册；首次部署/初始化在 **Web 管理台**或安装向导中**设置一次「管理员/教师」登录密码**（可再修改），凭此管理**课程与名单**。课堂场景、不涉及高价值隐私与资金的前提下，**不把**「教师账号被盗」按高优威胁建模（仍上 **HTTPS**；后续可加 2FA 等，非本期必项）。
- **角色**：仅**学生**与**教师**；教师即 **Web 管理端操作者**。

- **教学结构（无「作业系统」、无系统内打分）**  
  - **章**为学习单元。**章内**「**核心知识点 + 引导练习 + 扩展练习**」**首先**由 **AI 根据本人后续提供的章素材** **生成草稿**（见文首「章节内容来源与生成方式」），再经**教师审校**定稿入库。  
  - 每章拆为若干**「核心知识点」**块（顺排即可，不另建「作业批改域」的独立业务表，除非 `design` 为存储方便另有命名）。  
  - **每块**内**先**知识点说明（富文本/视频/外链在 `design` 定），**再**是**引导练习** cell：代码区含**可跑通**的提示/示例/半成品，学生以补全/运行成功为目标；**UI** 为 Notebook 风格（如 **In [ ]** cell、可执行代码区，布局对齐你提供的参考图）。  
  - **同一块**内，引导练习**之后紧接**一道**扩展练习** cell：题面**只**描述期望运行结果/行为；代码区**不**预置完整可抄的参考答案（可空 cell 或仅「在此编写」类注释），**不**在框内给标准解。  
  - 同一章内可有**多组**「核心点 + 引导 + 扩展」，由课决定。  
  - **通过规则（默认）**：各必做 cell（每块的引导+扩展，清单在 `design` 列清）在**本机/嵌入**执行**无未捕获异常**即该题**过关**；`design` 中可为扩展题加「期望输出/断言」等更严规则。学生在本章**全部必做过关**后，可**提交/完成本章**；**仅记章完成**（**无**教师打分）。**不**做「作业多版本评阅」；**可**只存**完成时间/状态**供教师看是否练完。

- **身份与学号防错号**：学号+姓名由教师**预导入**；学生**首次**注册/绑定时**须**与名单**完全一致**；**找回**不做邮箱/手机验证；**学生密码可逆**存储，教师可直读/重置（见前定）。

- **LLM Chatbot**：学生在**任一章、任意引导或扩展**过程中**随时**可打开/唤起 **Chatbot**；**不**做 RAG。国内 API 首发**暂缓**；密钥 **env/部署配置** 注入。对话是否落库在 `design` 定。

- **Notebook/执行**：默认 **JupyterLite / Pyodide**（§5）；需要任意 `pip` 或 C 扩展时切**本机 CPython 子进程**+可写 venv+**出站 `pip`（全开放）**。

# 前篇：整体准备清单（十段）

## 1. 产品与教学设计（非代码）

- **用户与场景**：学生用桌面**学章、跑 cell、过关、点完成**；教师用 **Web 编章、管名单、看完成进度**；**不**在系统内**打分**。
- **章内结构**（与参考图一致）  
  1. **核心知识点 + 引导练习 cell**：代码区有**可执行提示/示例/填空**，以 cell 中运行为成功。  
  2. **紧跟扩展练习 cell**：**文字**只给**运行结果/行为**要求；代码区**不**给完整可抄的解；学生可用 **Chatbot** 后自写再运行。  
  3. **Chatbot 全程可用**（不限定某一屏）。  
- **无**「作业成绩/教师录入分数/评语/导出成绩」**一期**范围。  
- **名单**：教师预导入学号+姓名；学生首登**须**与名单**完全一致**。

## 2. 唯一产品形态：学生桌面 + 教师 Web + 境外 API

- **学生端**：**Win/Mac 必装**；章内 **Notebook 风格**练习、逐 cell 过关、全章**完成/提交**；**Chatbot 与练习同应用内**（不强制开外站）。  
- **教师端**：**纯 Web、管理员密码、无注册**；**无**批作业/打分工作流。需**名单、章/知识点/引导+扩展 内容**（**JSON/简易 CMS，schema 在 `design`**）、**可**看学生各章是否完成。  
- **大模型/境外存数**：在 `design` 写清隐私与对话留存。

**逻辑分层**（简）：学生桌面 → **TLS** → **API**；教师 **Web**；**章**存库/文件；**执行**在本机/嵌入；**Chatbot** 调国内 **API**；库表：**用户/名单/章内容/章完成**（**无**评分子域）。

## 3. 账号与权限（无微信/QQ、无独立管理员）

- 学生注册登录；**教师**为**管理员密码**会话。`Role`：`student` / `teacher_session`。  
- 凭据、**可逆**学生密码、**无** OAuth：同**前文已定**。

## 4. 教学内容与练习编排（**非**作业/批改子系统）

- **编写流**：**输入** = **本人后续提供**的**章级原始内容**；**处理** = **AI 生成**结构化草稿（**核心点 + 引导题 + 扩展题**）；**输出** = 教师在 **Web** 中**编辑/通过**后写入**章、知识点块、每块内引导/扩展 cell**（`design` 定 **schema**）。**生成**与**发布**之间**必须**有人工确认（**可**记录 `generated_by_ai`、版本号，在 `design` 定）。  
- 可配**教学视频/外链**（如参考图）。若 **AI 草稿** 与 **JupyterLite/本机 pip** 能力不一致（例如生成了需 `pip` 但学生环境未开），**人审**时截住。  
- **过关与完成**：**无**教师批改队列。默认**无未捕获异常**即**题过**；**全章必做全过**后可**提交/标记章完成**；教师**只**看**完成/未完成**（**不**给分）。

## 5. Notebook / 代码执行：JupyterLite 与本机 CPython

- **UI**：Jupyter 风格（**In [ ]**、**Shift+Enter** 等）在 `design` 与参考图对齐。

| 场景 | 路线 |
|------|------|
| 标准库/小脚本、不依赖任意 `pip` | 优先 **JupyterLite / Pyodide** |
| 需 **PyPI 任意包**、C 扩展等 | **本机 CPython 子进程** + venv + **pip 全开放**（可配镜像、记安装日志） |

**共同底线**（`design` 量化）：子进程/沙箱、工作目录、CPU/内存、超时、**kill**、安装日志；**一期**不上 **JupyterHub 云**。

## 6. LLM Chatbot（无 RAG、练习中**随时**可用）

- 练习/扩展**任意时刻**可开 **Chatbot**；**可**附带**当前章、当前 cell** 元信息（`design` 定字段）。  
- 国内供应商**暂缓**；**HTTPS** 出站、限流、防注入、费用监控。

## 7. 工程与基础设施

- **CI**、学生端**签名**、教师 **Web+后端** 同仓常见。  
- **内容生产**：**批量/单章**调用**与章生成**同一或独立**模型 API**（**额度、延迟、重试**在 `design` 定）；**提示词**与**输出 JSON** 校验（**防**扩展题误带**完整标答**在 `starter_code`）在实现时落库或配置化。  
- 关系型库：**用户/名单/章/章完成/内容**；**不**将**成绩表**列为必建。  
- 监控：API、LLM 429/欠费、客户端崩溃。

## 8. 合规、安全与隐私

- 学号、章完成、对话、境外机；**可逆**密码、本机 **pip** 的课堂说明见前文；本系统**不**按等保/资损平台建模（可后加）。

## 9. 维护与排期（单人）

- 一人承担产品、教学、前后端、运维等，按里程碑自行切分。  
- **课稿节奏**：**各章原始内容**由**本人**按迭代提供；**AI 生成**与**人审定稿**时间**与 M1 内容量**一起排进计划。

## 10. 已定产品分叉（`design` 落表与 API）

- 学生桌面、教师 **Web+管理员密码**；**不**含打分。  
- 章内：**多组**「**核心点 + 引导 + 扩展**」。  
- 完成：全章**必做**过关 → **可**「提交/本章结束」；**只**记**章完成**（不评卷）。**若**存终稿代码仅为备份/复现，在 `design` 选做。  
- **JupyterLite/CPython、pip 全开放、可逆、国内 LLM 暂缓、预导入+匹配**同前文。  
- **章内容**：**后续提供素材** + **AI 生成** **核心点+引导+扩展** + **教师审校发布**。

# 续篇

## 续篇 1. 建议的交付阶段

| 阶段 | 目标 | 依赖准备（摘要） |
|------|------|------------------|
| M0 | 管理员密码、一名学生注册、一章**至少 1 组**引导+扩展、**cell 无未捕错误**判过、**Chatbot 能打开**一次、JupyterLite 或本机核其一 | 云、TLS、学生包、教师 **Web 壳** |
| M1 | 多章、多知识点块、**章完成**提交/游标、**至少一条**「**素材 → AI 生成 → 人审 → 发布**」闭环、教师**看谁哪章**完成、名单 | 内容 **schema**、**完成**状态表、**AI 生成** API 与**审校** UI |
| M2 | 限流/超时/pip 镜像、LLM 欠费、错误提示 | 小抄 |
| M3 可选 | 扩展题**输出/断言**、强沙箱；**JupyterHub** 仍非默认 | — |

**原则**：不上一期「批作业」、不默认上 **Jupyter 云**。

## 续篇 2. MVP 边界

- **MVP 必含**：练习过关、全章**完成提交**、**随练 Chatbot**、引导+扩展、**无打分**、名单+可逆等已定项；**至少一条**「**章素材 → AI 生成 → 教师审校 → 发布**」路径可跑。  
- **不做**：作业评分、成绩表、防抄袭、转班、微信/QQ、RAG、Jupyter 云（一期）。

## 续篇 3. 技术 spike

- 用**一段样例章素材**走通：**AI 生成** → 写入草稿 → **人改** → 落库为**多组**引导+扩展，扩展**无**可抄**完整** `starter_code`。  
- **Chatbot** 带**当前 cell id** 的 **E2E** 一次。  
- **本机** **CPython+pip 全开放** 跑一例如 **numpy**。

## 续篇 4. 风险

| 风险 | 说明 | 缓解 |
|------|------|------|
| **AI 命题**质量 | 错码、**扩展**误带**标答**、与运行环境不符 | **人审**必过、**JSON** 校验、**样章** 回归跑 cell |
| 扩展题仅「不报错」过松 | 空逻辑也过 | `design` **可**加期望输出/断言 |
| 教师单密码 | 课堂场景 | **HTTPS**；**可**后加 **2FA** |
| **pip** 供应链 | 全开放 | 日志、课堂规约 |
| 免费 LLM | 额度/价 | 监控、失败提示 |
| 境外存学号/对话 | 合规 | 告知、最小存 |

## 续篇 5. 已决与交给 `design` 的细目

**已决**（本清单**不再**开会争）：无作业/打分子系统；章=多组「核心+引导+扩展」；**扩展不内嵌完整参考答案**；**Chatbot 随时**；**完成=本机必做过关+可提交章完成**；**教师不评卷**；可逆/管理员/预匹配/国内 LLM 暂缓/pip 全开放见上文。

**`design` 须写清**：**cell** JSON、过关仅无异常**或**+输出/断言、**章完成**写库、终稿/对话**是否**存、可逆**加密与密钥**；**AI 生成** **Prompt**、**输入素材格式**、**输出与 schema 的映射**、**人审** **状态机**（草稿/已发布）。
