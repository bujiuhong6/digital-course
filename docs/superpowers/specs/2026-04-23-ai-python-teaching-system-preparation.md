## 当前优先架构假设（已定稿范围，变更时同步续篇 5 与决策记录表）

- **部署**：**境外轻量云主机**对外提供 **HTTPS**；**域名可在国内注册商购买**，DNS **A/AAAA 解析到境外 IP**（与「网站托管在大陆机房需 ICP 备案」不是同一套流程，具体以注册商与监管要求为准）。
- **客户端形态**：**学生必须安装** **Win / Mac 桌面端**；**教师使用纯 Web 管理台**（浏览器访问，**不强制**装学生端同名的桌面版）。后端对外 **HTTPS API** 同时服务**学生端与教师 Web**。
- **角色**：仅 **学生** 与 **教师**；**教师账号兼管**用户与内容（无独立管理员角色）。
- **作业与课程**：**一章一份作业**；**全班同一题目**；**无限次重做**；**不做防抄袭**；**无转班**业务。
- **身份与学号防错号**：**学号+姓名 由教师提前在 Web 管理台导入**（或批量上传）；**学生首次注册/绑定** 时**必须**输入**与名单完全一致的 学号 与 姓名**（**任意一项不符则注册失败**），以此防止输错、冒用；**找回密码不做邮箱/手机验证**；**教师可查看全班学生密码**，并具备**重置任意学生密码**能力（按校务安全要求控制教师端访问与审计）。
- **LLM Chatbot**：采用**尽可能低成本的接入**——优先评估**国内大模型 API 的免费/试用额度**（如阿里云通义、智谱 BigModel、百度文心等**以各平台当时官网规则为准**）；**不做 RAG**，对话以**当前练习上下文 + 可选系统提示**为主。
- **Notebook / 代码执行**：**默认**走 **JupyterLite / Pyodide**（见 §5）；**当课程需要**「**联网自行 `pip install` 装 PyPI 上任意包**」**或** 依赖 **C 扩展/非纯 Python 包** 等 Pyodide 难覆盖能力时，**切到 本机 CPython 子进程** + **可写 venv/用户目录** + **允许出站 `pip`（在客户端侧执行）**；**超时、资源上限、工作目录、恶意包与 supply-chain 风险** 在 `design` 中写清。

# 前篇：整体准备清单（十段）

## 1. 产品与教学设计（非代码）

- **用户与场景**：**学生、教师**两角色；**教师** 用 **Web** 建课、维护名单、发作业、批改、密码与内容管理；**学生** 用**桌面端** 学习与提交。
- **课程结构模型**：**章**；每章有 **教学内容** + **该章唯一一份作业**；全班题目相同；学生可**无限次重交**，保留**版本或仅保留最新**在 `design` 中二选一；**不实现转班、不实现防抄袭**。
- **名单与注册**：**教师** 在 Web 端 **预导入 学号+姓名**；**学生** 在桌面端 **自注册/首次绑定** 时 **严格匹配** 名单，**不匹配则无法通过**；**无**「学生随便填、教师后审」的宽松模式（除非你后续改产品）。
- **账号标识（桌面端）**：**学号** 与名单键一致；**桌面端** 本地草稿与**联网提交**、失败重试策略。
- **练习与执行**：见 §5；由平台**设定**单会话/CPU/内存与 **pip/网络** 策略。
- **评价与成绩**：以**教师给分+评语**为主；成绩单导出、迟交等规则在 `design` 中**按需**收敛。

## 2. 唯一产品形态：学生桌面 + 教师 Web + 境外 API

- **学生端**：本机 **Win/Mac 应用**（**必装**）；线下可编辑，联网 **HTTPS 提交**作业，按 **学号** 与 **章** 关联。
- **教师端**：**纯 Web 管理台**（**浏览器**）；**不强制** 安装与学生相同的桌面版；需支持：**名单导入、章节内容、作业题面、按学号查看提交与历史重做、打分、查看/重置学生密码**。
- **大模型数据流**：题面与学生提问是否发往模型、对话是否**落库**、教师是否**审计对话**，在**隐私与校务**条款中写清；数据存**境外**时与**教育数据、出境**要求一并评估。

**典型逻辑分层**：学生 **桌面** → **TLS** → **API**；教师 **浏览器** → **同一 API / 同域 Web**；**关系型/对象存储**；可选 **JupyterLite 内嵌** 或 **本机 CPython 子进程**；**Chatbot** 经**国内 API 网关**（见 §6）；作业与元数据落库。

## 3. 账号与权限（无微信/QQ、无独立管理员）

- **身份模型**：`Role` ∈ {`student`, `teacher`}；**教师** 拥有**名单、用户与作业管理、密码查看与重置、内容发布**等能力；**不单独建 admin 表** 亦可（权限绑定 `teacher`）。
- **凭据与注册**：**学号+姓名 预导入** + **首登严格匹配**（见上）；**学号+密码** 用于日常登录；**无** 邮箱/手机作为找回通道；**教师** 可 **看/设/重置** 学生密码，实施上需 **强审计、HTTPS、教师账号强保护**。
- **不使用** OAuth 与微信/QQ 登录。

## 4. 教学内容与作业子系统

- **内容管理**：章级 **Markdown/富文本**、代码块与附件；**仅教师** 可编辑发布（Web）。
- **作业系统**：**每章一题、全班同题**；**无限重做**；提交物规格在 `design` 中定；**教师 Web** 批改；学生桌面 **离线队列** 定义清楚。

## 5. Notebook / 代码执行：JupyterLite 与本机 CPython 的分界

- **目标**：在**小体量教学**下**最快落地**；**学生联网可自行 `pip install`（对 PyPI）** 时，以 **本机 CPython 子进程** 为主能力承载，**JupyterLite/Pyodide** 为**轻量默认/补充**（适合标准库+少数 micropip 场景）。

**何时用 JupyterLite / Pyodide（优先简单、无系统级 pip 依赖）**  
- 以 **标准库**、教学脚本为主，**不依赖** 需从 PyPI **任意**拉取的**重型/二进制** 包。  
- 或作为 **入门章节** 的快速演练层（**启动快、包体可预测**）。

**何时切到 本机 CPython 子进程（联网 pip）**  
- **需要** 学生执行 `pip install <包名>` 安装**任意**（或**课程指定列表外**）的 **PyPI 包**；或依赖 **C 扩展、非 wheels 在 Pyodide 不可用** 的库。  
- 实现上：**内嵌/随包携带** 某版本 **CPython**、**每生/每作业 venv 或 用户级 site-packages 目录**、**允许 HTTPS 出站到 PyPI 与** 选定的 **index URL**、**可配置镜像**（如清华/阿里源）、**记录安装日志** 便于排错。

**共同底线**（`design` 中量化）：**子进程/容器化隔离**、**工作目录**、**CPU/内存/单会话时长**、**子进程** **kill**、**对危险 `pip` 包名或脚本** 的**可选黑名单**、**供教师审计的安装记录**；**JupyterHub 集群** 仍**不作为** 一期目标。

## 6. LLM Chatbot 子系统（免费优先、无 RAG）

- **产品范围**：**不建向量库、不做 RAG**；上下文以**当前题/当前章节 + 用户当轮输入** 为主；是否附带**代码片段** 在 UI 中可选。
- **「免费」落地方式**：在**国内**各开放平台注册，使用其**新客/限免/试用 token** 档（**通义、智谱、文心** 等，**以申请时官网为准**）；超预算时按量/限流/降级。
- **技术**：**HTTPS 出站** 至所选 API 域名；**流式/非流**、**限流**、**关键审计**、**防注入**、费用监控。

## 7. 工程与基础设施

- **代码与协作**：单仓优先；**CI**；**学生** **Win/Mac 构建与代码签名**；**教师** **Web 前端 + 同仓后端**。
- **运行与部署**：**境外轻量云**、**Let’s Encrypt** 或云证书、**WAF/安全组** 仅开必要口；**关系型库** 存用户、**预导入名单**、章、作业、提交、**注册绑定状态**、**可选**对话摘要。
- **域名**：国内购域 + 解析境外；**备案** 与**大陆托管** 关系见假设节。
- **监控告警**：**API 错误、LLM 429/欠费、学生端崩溃率**（若可采集）。

## 8. 合规、安全与隐私

- **隐私与校务**；**未成年人** 如适用；**教育数据** 与**境外机** 存储的告知与同意；**学生自行 pip** 属于 **供应链** 与 **本机** 安全边界，在条款与**课堂规范**中说明。
- **教师可见学生密码** 为**高敏** 能力，需**共识** 与**访问留痕**；**学生 pip** 需**可观测** 与**异常提示**（至少教师可见安装记录，可选）。

## 9. 团队与项目治理

- **角色**：产品、教学、**学生客户端**、**教师 Web**、后端、**运维** 可兼岗。
- **预算**：**境外主机**、域名、**LLM 超额** 缓冲、**学生端代码签名**；**大集群 Notebook 云** 非一期必项。

## 10. 已定产品分叉（已写入上文；`design` 中落到接口与表结构）

- **教师** **仅 Web**；**学生** **必装桌面**；**教师** **可选用浏览器** 完成全部管理（**不装学生端**）。
- **JupyterLite** 与 **本机 CPython+联网 pip** 的**选型**：见 **§5**；`design` 中列出 **课纲依赖包** 与**默认**走哪条执行栈。
- **学号+姓名** **仅教师预导入**；**学生** **首登输入必须完全一致** 否则**拒绝注册/绑定**。

# 续篇：分阶段、MVP、风险、待澄清

## 续篇 1. 建议的交付阶段（从准备到可运营）

| 阶段 | 目标 | 依赖准备（摘要） |
|------|------|------------------|
| M0 概念验证 | **名单导入** + 学号**匹配**注册 + **HTTPS** + 一章一作业 + **一次** Chatbot + **JupyterLite 或** **CPython+pip 最小闭环** | 境外云、域名、TLS、国内 API 密钥、学生端 Hello 包、**教师 Web** 一屏 |
| M1 教学端 MVP | **多章**、**无限重做**、**教师打分**、**看/设密码/重置**、**pip+CPython 路径** 按课纲能跑通 | 库表、教师权限、**审计**、**名单与绑定** 状态机 |
| M2 体验加固 | 学生端更新、**限额/超时/ pip 镜像** 打磨、**LLM 欠费** 策略、**日志与备份** | 运营手册 |
| M3 可选加强 | **更强沙箱**、**对 pip 的治理**（黑名单/审批）若教学需要 | 不扩 RAG、不上 JupyterHub 除非**明确**需要 |

**分期原则**：**别在一期** 堆 **JupyterHub 云**；**先** 跑通 **名单 + 学生端 + 教师 Web + 一条能 pip 的 CPython 链**（若课纲需要）。

---

## 续篇 2. MVP 与「全量」的边界

- **MVP 必含**：两角色、**预导入+匹配注册**、**一章一作业、同题、无限重做**、**学生桌面 + 教师 Web**、**教师看/重置密码**、**HTTPS**、**能 pip 的 CPython 路径**（课纲有要求时）+ **JupyterLite 可选**、**无 RAG** 的 **Chatbot**。
- **刻意不做**：**微信/QQ**、**防抄袭**、**转班**、**多管理员**、**RAG**、**云端 JupyterHub 教学**（一期）。

---

## 续篇 3. 技术选型 spike（轻量）

- **学生** **Electron 或 Tauri** + 后端 + **教师 Web** 同域联调。
- **JupyterLite** 一屏 + **本机** **CPython** **子进程+venv+pip** 一屏，各跑通同一章节的**依赖差异** 样例（如「仅标库」vs「需 `pip install numpy`」）。
- 选定 **1 家** 国内 LLM API，读清**免费额度、QPS**；写 **fallback** 文案。

## 续篇 4. 风险登记

| 风险 | 说明 | 缓解 |
|------|------|------|
| 教师通览密码 | 教师账号**被盗** 则影响大 | 教师**强身份**、**二因素**（仅教师可开）、**操作审计**、**HTTPS** |
| 学生 **pip** | **恶意/投毒包**、**本机** 与**供应链** 风险 | **日志**、**可选黑名单/允许列表**、**课堂规范**、**教师可见** 安装历史 |
| 免费 API 不可持续 | 额度用尽、变价 | 用量与费用监控、**学生侧** 明确提示 |
| 数据在境外机 | 学号、作业、对话 | 告知、校务、**最小** 留存 |
| 名单**泄露** 或**导入错行** | **错绑** 或**冒名** 风险 | **导入校验**、**样例** 对账、**首登** **严格** 学号+姓名（已定） |

## 续篇 5. 待澄清问题（尚需在 `design` 中写死项）

1. **学生密码的存储形态**：**可逆**（为「教师可见」）** vs** **不可逆哈希 + 仅重置**？若必须**看见明文**或「代登」，在 `design` 中写清 **加密与密钥** 与**谁**可解密。
2. **重做**：保留**每次提交** 还是 **只保留最新**？影响存储与教师批改 UI。
3. **国内 API** 三选一**首发** 哪一家？（在 spike 后填决策表。）
4. **pip 策略**：`pip` 是否**全开放 PyPI** 还是**仅** **教师白名单** 包/版本？与 **§5** 强相关。
